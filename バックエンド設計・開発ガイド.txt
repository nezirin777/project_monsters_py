バックエンド設計・開発ガイドライン（品質保守指標）

本ドキュメントは、リファクタリングが完了したPython/CGIバックエンドコードの一貫性、保守性、および型安全性を永続的に維持するためのコーディング規約および品質管理指標です。
1. 基本方針

    機能のシンプルさと可読性の重視
    新規機能や修正を加える際は、コードベースを不必要に複雑化させないことを最優先とします。

    要所における「なぜそうしたか」のコメント付与
    コードの動作（What）だけでなく、その設計を選んだ理由（Why）をコメントとして残すことを義務付けます。

    型チェックと静的解析の継続的なクリーン維持
    型チェッカー（MypyやPyrightなど）や静的解析ツールで警告がゼロである状態を保ちます。

2. 徹底すべきコード設計基準（アンチパターンの再侵入防止）

今後コードを修正・追加する際は、以下のアンチパターンを決して混入させないでください。
2.1. NoReturn 関数の取り扱い

error()、success()、info()、print_html()、sys.exit() など、呼び出し後にプロセスを終了するか例外を投げる関数はすべて NoReturn（制御が戻らない）関数です。

    戻り値（return）の排除
    これらの関数を呼び出す際、return error(...) のように記述してはいけません。必ず単独で呼び出します。

    デッドコード（到達不能領域）の明示
    NoReturn 関数の直後にある return や return None などのデッドコードは削除するか、型チェッカーにその旨を伝えるため raise RuntimeError("unreachable") に置き換えてください。

2.2. ミュータブルデフォルト引数の禁止

デフォルト引数に辞書（{}）やリスト（[]）などの変更可能なオブジェクトを直接指定することは、プロセス内で状態が共有・蓄積され、予期せぬバグを引き起こすため禁止します。

    正しい記述:
    code Python

    def process_data(x: dict | None = None) -> None:
        x = x or {}

2.3. None安全（None-Safety）の確保

引数が None のまま処理が進行してランタイムエラーを起こすような関数は作成しないでください。

    引数を必須（None を許容しない）にする。

    引数に None を許容する場合は、必ず関数の冒頭で None ガード（早期リターンや例外送出、フォールバック値への置換）を追加する。

2.4. リスト線形探索の回避（O(1) メンバーシップテストの徹底）

特定の値がグループ内に含まれるかを判定する際、大規模なリストや動的なジェネレータを用いた線形探索（O(N)）を日常的に記述することを避けてください。

    探索対象が固定されている、または限定的である場合は、モジュールレベルで frozenset（不変集合）として定義し、定時間（O(1)）での照合を徹底します。
    code Python

    # モジュールレベルで一度だけ定義
    _ALLOWED_MODES: frozenset[str] = frozenset({"mode_a", "mode_b", "mode_c"})

    if x in _ALLOWED_MODES:
        ...

2.5. 未使用変数のクリーンアップ

    例外オブジェクトの束縛
    例外処理（except 節）で、変数（as e）を参照していないにもかかわらず定義したままにしないでください。
    code Python

    # 推奨
    except Exception:
        do_something()
        raise

    モジュールレベル・ローカルの未使用変数
    定義されているが一度も参照されない変数は、確認のうえ速やかに削除してください。

2.6. ルーティングにおけるデフォルトルート（else節）の保証

if / elif によるモード分岐やディスパッチ、あるいは判定処理を行う関数には、必ず予期しない入力に備えたフォールバック用の else 節を設け、エラーハンドラを呼び出してください。無言で None を返して素通りさせてはいけません。
3. 型ヒント（Type Hints）と PEP 8
3.1. 必須型ヒントとモダナイズ

    アノテーションの必須化
    新しく定義する関数、およびメソッドの引数・戻り値には、必ず型ヒントを記述します。

    モダンな型表記（Python 3.10+ 準拠）

        Optional[str] や Union[str, int] のような古い表記は避け、str | None や str | int のように |（垂直バー）演算子を用いて簡潔に記述します。

        List や Dict などのジェネリック型も、大文字（typing.List）ではなく組み込みの小文字表記（list, dict, tuple）を使用します。

    マッピング構造への明示
    関数マップや動的定義オブジェクトにも型ヒントを付与します。
    code Python

    FUNCTION_MAP: dict[str, tuple[str, str]] = { ... }

3.2. PEP 8 とエントリポイント制御

    sys.stdout.reconfigure の責務分離
    ライブラリモジュール（sub_def/ 配下など）の中で sys.stdout.reconfigure(encoding="utf-8") を呼び出してはいけません。文字コード構成は、直接実行されるエントリポイント（login.py, monster.py など）のみに一任します。また、エントリポイント内での重複記述（モジュールレベルと __main__ 内の双方での呼び出し）も避けてください。

    インポートスタイルの統一
    1行に複数のモジュールをまとめてインポート（import sys, os）することを禁止します。1行1モジュールに分割して記述してください。

4. コメント記述基準（「なぜ」の明文化）

コード自体の意味をなぞるだけのコメントではなく、**「設計上の意図」や「特殊な制約」**を説明するコメントを必ず記述してください。

    関数内インポートの意図
    循環インポート（Circular Import）を回避するために敢えて関数内でインポートしている場合は、その旨をコメントで明記します。

    raise RuntimeError("unreachable")
    NoReturn 関数の後などで「静的型チェッカー向けに到達不能であることを通知するため」に記述していることを明記します。

    安全装置・ガードの目的
    再入防止や多重リダイレクト、あるいは万が一処理が素通りした場合のフェイルセーフとして記述された制御コードには、その目的（例: 「万が一の素通りに備えた安全網」）を付記します。

    複雑な、または特殊な処理
    Shift-JISへの変換・置換比較、タイミング攻撃対策を考慮した定時間比較（hmac.compare_digest）など、一見して不自然または複雑に見える処理には、その論理的背景を記述します。

    意図的なスキップ・正常系としてのフォールバック
    ロックの取得失敗によるスキップや、二重実行防止のガードなど、例外を投げずに処理を終了する箇所には、「エラーではなく正常系としての挙動である」ことを明記します。

5. 健全性維持のための変更時チェックリスト

コードを改修した際は、リクエストをコミットする前に以下のセクションに沿ってセルフチェックを行ってください。

残留コメントのクリーン化
開発過程で残された草稿用のメモ（# 変更前 や # TODO など）が残っていないか。

global 宣言変数の生存期間の確認
global 宣言されているにもかかわらず、実際にはローカルスコープ内でしか使われていない、あるいは書き込みだけで読み取りが行われていない変数がないか。

    不要な戻り値の最適化
    呼び出し側で戻り値を一切使っていないにもかかわらず、関数が不要な値を返していないか（不要な場合は戻り値を None に最適化します）。

6. パフォーマンスと I/O キャッシュ指標

CGIは「1リクエスト1プロセス」の特性を持ち、永続的なアプリケーションキャッシュ機構を持ちません。しかし、**「同一リクエスト（1プロセス）内で、複数回呼び出される可能性のある重いI/O」**については、モジュールレベルでのキャッシュを徹底します。

    静的データ・ゲーム内不変データのキャッシュ化
    性格データ（seikaku_dat）や特技定義（tokugi_dat）など、ゲーム中に内容が変わらないマスタデータについては、関数が呼ばれるたびに毎回ファイル（PickleやCSV）からロードするのではなく、モジュールレベルでキャッシュ変数に1回だけ読み込んで利用する仕組みを維持・拡張してください。
    code Python

# モジュールロード時に一度だけ読み込む
_SEIKAKU_KEYS: list[str] = list(open_seikaku_dat().keys())
