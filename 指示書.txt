# 方針

昔のCGIゲームらしい「情報量の多いブラウザゲーム感」を残しつつ、
現代的なUI/UXと保守しやすいCSS設計へ段階的に改善してください。

目的は「今風のSPA」ではなく、
「古き良きCGIゲームを現代的に再構築すること」です。

世界観・密度感・ゲームっぽさを重視してください。


# デザイン思想

- 淡い寒色系UIをベースにする
- 白ではなく slightly blue-gray な背景色を多用
- カードUI主体
- 軽いグラデーションはOK
- 過度なフラットデザインにはしない
- 少しレトロゲーム感を残す
- 「個人運営CGIゲームの進化系」みたいな雰囲気を目指す

避けるもの：

- 真っ白ベタ背景
- 原色だらけ
- ミニマルすぎるデザイン
- スマホアプリ風UI
- 意識高いSaaS感
- Apple風デザイン
- 極端なモダンUI


# CSS設計方針

CSSは「後から触れる構造」を最優先してください。

重要：

- レイアウトと見た目を分離
- 共通コンポーネント化
- modifier設計
- スコープを閉じる
- ページ固有classを分離

推奨：

- .card
- .btn
- .badge
- .grid
- .section

などの共通ベースを作る。

modifier例：

- .card--battle
- .card--shop
- .btn--danger
- .btn--wide

BEM寄りの設計歓迎。

例：

- .rank-card
- .rank-card__header
- .rank-card__body

避けるもの：

- ページ跨ぎの副作用
- セレクタ競合
- 同じclassの再定義乱発
- 巨大な複合セレクタ
- !important頼り


# HTML方針

- section単位で役割を分離
- card-header/card-body構造を基本化
- 情報をカード単位で整理
- 縦長になっても見やすく
- anchorジャンプや固定メニュー歓迎

UIは「ゲーム画面」として設計してください。


# UX方針

重要：

ユーザーは長時間プレイする。

なので：

- 目が疲れにくい
- 情報が探しやすい
- どこを押せるかわかる
- スクロール負荷を減らす

を重視してください。

リンクは必要に応じてボタン化。

長いページには：

- フロートメニュー
- セクションジャンプ
- sticky UI

歓迎。


# モンスター表示方針

モンスター画像はゲームの主役。

- 画像縮小しすぎ禁止
- 縦長モンスターを潰さない
- max-height重視
- max-height:230px,max-width:110pxほど。
- カードよりモンスターを優先

「UIのためにモンスターを犠牲にしない」。


# 既存コードへの対応

既存コードは古いCGIゲーム由来。

そのため：

- 一気に作り直さない
- 段階的改善
- 既存テンプレ構造を活かす
- 小さく整理していく

方向で提案してください。

全面React化みたいな提案は不要。


# 提案時のルール

提案は：

- 実際のHTML構造を見て判断
- 既存classとの整合性を考慮
- 既存のコメント行はなるべく維持
- コピペしやすく
- すぐ試せる形で


出してください。

可能なら：

- 「どこに追加するか」
- 「どこを置き換えるか」
- 「なぜそうするか」

も説明してください。


# 特に重視すること

- 保守性
- 統一感
- ゲームっぽさ
- 情報密度
- 長時間見ても疲れないこと
- 「古き良きブラゲ感」

を維持してください。
